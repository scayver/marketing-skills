- Write between 1,500 and 10,000 words per full deliverable unless the user asks for a shorter artifact.

- Mobile-friendly: short paragraphs, clear headers, scannable tables.
- Practical and actionable. Every section connects to a next step.

## What This Skill Does

This skill turns the library into a marketing operating system. It does not replace the other skills. It routes work to them, defines the order of operations, connects strategy to execution, and creates a repeatable cadence for planning, shipping, measuring, and improving marketing.

Use this skill first when the user asks for a broad growth plan, a marketing roadmap, a full launch system, or a way to coordinate multiple skills and tools.

A Marketing OS has five jobs:

1. Define the market and message.
2. Select the growth motions.
3. Build the assets and campaigns.
4. Connect the tools and data.
5. Run a weekly improvement loop.

Next step: identify the business stage before choosing workflows.

## Product Context Readiness Gate

Before you run any Marketing OS workflow, check whether the user has usable product context.

Read or request these minimum details:

- Product name and category.
- ICP or buyer segment.
- Primary offer or conversion goal.
- Core pain, desired outcome, and main objection.
- Proof points, constraints, or known assumptions.

If `.marketing-os/product-context.md` exists but still contains blank template fields, treat that as a blocker. Do not write campaign assets, outreach, landing page copy, SEO briefs, tool recommendations, or a full execution plan yet.

Instead, route the user to `product-marketing` and complete the missing context first. Ask one focused question at a time if the user has not provided enough detail.

When context is ready, summarize the usable assumptions in 5 bullets before continuing the active workflow.

Next step: confirm product context before producing the first deliverable.

## OS Layers

Use these layers to organize the work.

| Layer | Purpose | Primary Skills | Tool Support |
|-------|---------|----------------|--------------|
| Strategy Layer | Decide who you serve, what you promise, and why you win | `product-marketing`, `business-strategy`, `customer-research`, `brand-storytelling`, `marketing-ideas` | Research, CRM, website, docs, competitor tools |
| Demand Layer | Create awareness and qualified demand | `seo-content-strategy`, `seo-blog-writer`, `social`, `linkedin-strategy`, `pr-communications`, `ads`, `directory-submissions` | SEO tools, social tools, ad platforms, PR lists |
| Conversion Layer | Turn attention into leads, trials, buyers, or demos | `landing-page-cro`, `copywriting`, `sales-copywriting`, `cro`, `signup`, `pricing`, `paywalls` | Analytics, heatmaps, A/B testing, forms, payment tools |
| Pipeline Layer | Turn leads and accounts into sales conversations | `prospect-research`, `cold-email`, `sales-enablement`, `revops`, `email-marketing` | GitHub, Apollo, ZoomInfo, HubSpot, Salesforce, Hunter, Clay |

## Operating Cadence

Use this cadence to run the Marketing OS.

### Weekly

- Review the scoreboard.
- Pick one priority growth motion.
- Ship one campaign or asset.
- Fix one conversion bottleneck.
- Repurpose one piece of content.
- Add one learning to the product marketing context.

### Monthly

- Review channel performance.
- Refresh ICP and positioning assumptions.
- Audit SEO, conversion, email, and pipeline quality.
- Update the experiment backlog.
- Decide which workflows should continue, pause, or scale.

### Quarterly

- Reset the growth strategy.
- Choose the primary operating motion for the quarter.
- Define campaign themes.
- Set target metrics.
- Update tooling and data requirements.
- Create the 90-day execution roadmap.

## Core Workflows

### SaaS Launch Workflow

Use when the user wants to launch a SaaS, feature, open-source project, AI tool, or agent product.

Order:

1. `product-marketing` for positioning and ICP.
2. `business-strategy` for business model and customer journey.
3. `launch` for pre-launch, launch day, and post-launch plan.
4. `landing-page-cro` for the launch page.
5. `email-marketing` for waitlist, launch, and follow-up emails.
6. `linkedin-strategy` and `social-content-planner` for launch content.
7. `pr-communications` for press and founder announcement.
8. `analytics` for tracking and attribution.

Tool references:

- `github` for repo and developer launch signals.
- `ga4` for launch traffic.
- `mailchimp`, `klaviyo`, `activecampaign`, or `customer-io` for launch email.
- `hubspot`, `salesforce`, or `pipedrive` for lead capture.

Next step: create a launch scoreboard with visitors, signups, activation, replies, demos, and revenue.

### SEO Growth Workflow

Use when the user wants organic traffic and topical authority.

Order:

1. `product-marketing` for positioning and ICP.
2. `customer-research` for pain points and language.
3. `seo-content-strategy` for keyword clusters and briefs.
4. `seo-blog-writer` for long-form posts.
5. `schema` for structured data.
6. `ai-seo` for AI answer visibility.
7. `content-repurposing` for social and newsletter assets.
8. `analytics` and `seo-audit` for measurement.

Tool references:

- `ahrefs`, `semrush`, `dataforseo`, `google-search-console`, and `ga4`.

Next step: build a topic cluster map before writing individual posts.

### Outbound Pipeline Workflow

Use when the user wants leads, prospects, sales meetings, or account research.

Order:

1. `product-marketing` for ICP.
2. `prospect-research` for account lists and intent signals.
3. `sales-enablement` for account briefs and talk tracks.
4. `cold-email` for email sequences.
5. `linkedin-strategy` for connection and DM support.
6. `revops` for lead stages, CRM fields, routing, and attribution.
7. `analytics` for reply, meeting, and pipeline reporting.

Tool references:

- `github`, `apollo`, `zoominfo`, `hunter`, `clay`, `hubspot`, `salesforce`, `close`, `pipedrive`.
- CLI: `github-prospects.js`, `apollo.js`, `hunter.js`, `zoominfo.js`.

Next step: require a trigger or intent signal before writing outreach.

### Conversion Workflow

Use when the user has traffic but not enough signups, demos, orders, or upgrades.

Order:

1. `analytics` for event and funnel diagnosis.
2. `cro` for friction analysis and hypotheses.
3. `landing-page-cro` for page rewrite and layout flow.
4. `copywriting` or `sales-copywriting` for sharper messaging.
5. `ab-testing` for test design and sample size.
6. `pricing`, `signup`, or `paywalls` for specific conversion moments.

Tool references:

- `ga4`, `hotjar`, `mixpanel`, `amplitude`, `optimizely`, `google-search-console`.

Next step: identify the highest-value conversion event before changing copy.

### Retention Workflow

Use when the user wants lower churn, stronger onboarding, or more advocacy.

Order:

1. `analytics` for cohort and activation analysis.
2. `onboarding` for time-to-value.
3. `churn-prevention` for cancellation and dunning.
4. `customer-service` for support templates and help center content.
5. `emails` or `email-marketing` for lifecycle campaigns.
6. `referrals` and `community-marketing` for advocacy.

Tool references:

- `intercom`, `freshdesk`, `zendesk`, `customer-io`, `mailchimp`, `mixpanel`, `amplitude`, `discord`, `slack`.

Next step: define the activation milestone and churn reason categories.

## Decision Rules

Use these routing rules.

- If `.marketing-os/product-context.md` is blank or generic, pause execution and run `product-marketing`.
- If the user lacks positioning, start with `product-marketing`.
- If the user lacks a business model or offer, use `business-strategy`.
- If the user lacks audience proof, use `customer-research`.
- If the user needs traffic, choose an acquisition workflow.
- If the user has traffic but weak conversion, choose the conversion workflow.
- If the user has leads but weak sales follow-up, choose the pipeline workflow.
- If the user has customers but churn, choose the retention workflow.
- If the user has scattered content, use `content-repurposing`.
- If the user needs public launch momentum, use `launch`, `pr-communications`, and `linkedin-strategy`.

Next step: do not run every skill. Pick the smallest set that moves the current bottleneck.
