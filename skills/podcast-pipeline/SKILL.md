# Podcast Pipeline

## Mandatory Content Standards

- Write in active voice.
- Use short sentences and plain language.
- Address the user as "you" and "your."
- Use `.marketing-os/product-context.md` when it exists. If it is blank or missing, recommend running `product-marketing` first.
- Do not invent guest quotes, timestamps, episode claims, statistics, or sponsor copy.
- Preserve meaning when summarizing transcripts.
- Mark unclear transcript sections as "needs review."
- Respect copyright, guest approvals, platform rules, and brand voice.
- Do not use em dashes, hashtags, emojis, or filler closings.
- End every full deliverable with one specific next step.

## Objective

Use this skill to convert one episode into a content system. The output should help you publish faster, keep the guest accurately represented, and produce assets for search, social, email, and sales enablement.

Next step: collect the transcript, episode title, guest name, and target channels.

## Pipeline Stages

| Stage | Output |
|-------|--------|
| Extract | Themes, claims, stories, quotes, examples, objections |
| Package | Title options, episode hook, description, show notes |
| Repurpose | Blog, newsletter, social posts, carousel, clips |
| Distribute | Platform-specific CTAs and publishing order |
| Measure | Reach, saves, comments, clicks, replies, subscribers |

Next step: identify the strongest episode theme before creating assets.

## Source Extraction

Pull these elements from the transcript.

| Element | What To Capture |
|---------|-----------------|
| Big idea | The main lesson or argument |
| Audience pain | Problem the listener recognizes |
| Story | Personal example, customer example, or turning point |
| Useful quote | Short quote that carries meaning without context |
| Tactical advice | Steps listeners can apply |
| Contrarian angle | A useful belief that challenges common advice |
| CTA tie-in | Natural bridge to offer, newsletter, product, or community |

Do not polish quotes beyond light cleanup unless the user asks for paraphrasing.

Next step: choose the three most reusable moments.

## Output Format

Use this table.

| Asset | Hook | Core Copy | Visual Direction | CTA | Source Timestamp | Approval Note |
|-------|------|-----------|------------------|-----|------------------|---------------|

For show notes, use this structure:

1. Episode summary.
2. Why it matters.
3. Key takeaways.
4. Notable quotes.
5. Resources mentioned.
6. CTA.

For clips, use this structure:

| Clip | Timestamp | Hook | On-Screen Text | Caption | CTA |
|------|-----------|------|----------------|---------|-----|

End with a 7-day publishing schedule.
