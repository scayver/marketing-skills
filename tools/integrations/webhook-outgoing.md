# Outgoing Webhook

Outgoing webhooks send HTTP POST requests to an external URL when an event occurs, delivering structured JSON payloads to any endpoint that can receive them.

## Capabilities

| Integration | Available | Notes |
|-------------|-----------|-------|
| API | ✓ | Sends HTTP requests to any endpoint |
| MCP | - | Not applicable |
| CLI | - | Not applicable |
| SDK | - | Not applicable |

## Authentication

- **Type**: Various (configured per webhook)
- **Header**: `Authorization: Bearer {token}`, `X-API-Key: {key}`, or custom header
- **Get token**: Generated by the receiving service; configure in webhook settings

## Common Agent Operations

### Send a basic webhook payload
```
POST https://your-endpoint.com/webhook

Content-Type: application/json
Authorization: Bearer {token}

{
  "event": "form_submitted",
  "timestamp": "2024-01-15T10:30:00Z",
  "data": {
    "name": "Jane Smith",
    "email": "jane@example.com",
    "message": "Hello"
  }
}
```

### Send with HMAC signature verification
```
POST https://your-endpoint.com/webhook

Content-Type: application/json
X-Signature: sha256={hmac_signature}

{
  "event": "order_completed",
  "order_id": "12345",
  "amount": 99.99
}
```

### Send with retry metadata headers
```
POST https://your-endpoint.com/webhook

Content-Type: application/json
X-Retry-Count: 0
X-Webhook-ID: wh_abc123

{"event": "user_registered", "user_id": 42}
```

## Key Fields

### Webhook Payload
- `event` - Event type that triggered the webhook
- `timestamp` - ISO 8601 time of the event
- `data` - Object containing event-specific fields
- `id` - Unique identifier for this webhook delivery

### Configuration
- `url` - Destination endpoint URL
- `method` - HTTP method (typically POST)
- `headers` - Custom headers to include
- `retry_count` - Number of retry attempts on failure
- `timeout` - Request timeout in seconds

## Parameters

- `Content-Type` - Should be `application/json` for most endpoints
- `Authorization` - Auth header value as required by the receiving service
- `payload` - JSON body containing event data

## When to Use

- Delivering real-time event notifications to external services
- Integrating with platforms that accept webhook ingestion (Zapier, Make, n8n)
- Syncing data to CRMs, marketing tools, or data warehouses on event triggers
- Notifying external APIs of ecommerce, form, or registration events
- Triggering serverless functions or microservices from site activity

## Rate Limits

- Depends entirely on the receiving endpoint's limits
- Implement retry logic with exponential backoff for 429 or 5xx responses
- Standard timeout recommendation: 30 seconds per attempt
