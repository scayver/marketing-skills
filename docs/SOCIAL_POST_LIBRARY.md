## Posting Strategy

Use this content as a 90-day launch and education campaign.

| Week Type | Content Mix | Goal |
|-----------|-------------|------|
| Launch week | Mixed category posts, top skills, GitHub prospecting CLI | Drive stars and installs |
| Education weeks | One skill post per day plus one integration post | Show practical use cases |
| Tool weeks | CLI posts, API guide posts, workflow examples | Attract builders and technical marketers |
| Proof weeks | Demo posts, before/after workflows, category stacks | Show why the repo is useful |

## Recommended Weekly Rhythm

| Day | Post Type |
|-----|-----------|
| Monday | Category stack post |
| Tuesday | Individual skill post |
| Wednesday | Integration guide post |
| Thursday | CLI tool post |
| Friday | Demo or workflow post |
| Saturday | Founder/build-in-public post |
| Sunday | Recap and install CTA |

- Star the repo: https://github.com/scayver/marketing-skills.
- Browse the tools registry: `tools/REGISTRY.md`.
- Try the GitHub prospecting CLI: `node tools/clis/github-prospects.js prospects export --repo owner/repo --dry-run`.

## Platform Notes

### LinkedIn

Use the `hook`, `post`, and `cta` columns together. Keep code snippets in backticks. Add a first comment with the repo link if you want the main post cleaner.

### X

Use the `hook` as the first post. Use the `post` as the second post. Use the `cta` as the final post in the thread.

### Threads

Use the `hook` as the opener. Break the `post` into two short paragraphs. End with the install command.

### GitHub Discussions

Use category stack posts as discussion starters. Ask builders which skill or integration they want next.
